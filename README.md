CLI Cafe Ordering System ☕
I built this project to explore how a simple terminal interface can handle the real-world logic of a busy cafe. Instead of just writing code, I wanted to focus on the user’s flow—from browsing a dynamic menu to receiving a clean, itemized receipt.

Why I Created This
It seems to me that the best way to master Python is to solve a practical problem. In this project, I prioritized:

User Experience: Designing a CLI that feels intuitive and doesn't crash on "fat-finger" input errors.

Logic Flow: Managing a "cart" system that accurately tracks quantities and prices.

Clean Code: Keeping the menu data separate from the ordering logic for easy updates.

Features
Dynamic Menu: Easily update items and prices in the source code.

Interactive Cart: Add, view, and confirm orders before finalizing the bill.

Formatted Receipts: Generates a clear breakdown of the total cost.

Input Validation: Built-in checks to ensure the system handles unexpected user input gracefully.
